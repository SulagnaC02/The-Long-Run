# Future Features

## 1. Intelligent Praise Engine

### Philosophy

The AI should never use generic praise or motivational messages.

Praise should only be given when there is objective evidence of meaningful progress.

Examples:

* Breaking a long-term negative pattern.
* Recovering after a difficult week.
* Maintaining consistency over time.
* Completing an important task despite genuine obstacles.
* Reaching a milestone.

Examples of good praise:

> "Two weeks ago you consistently postponed DSA. This week you completed four study sessions. That's genuine progress worth recognizing."

Examples of bad praise:

* "Amazing job!"
* "You're doing great!"
* "Keep it up!"

without supporting evidence.

The goal is to build trust through honesty rather than constant encouragement.

---

## 2. Adaptive Reflection

Reflection should feel like a short conversation rather than a survey.

Instead of asking for a satisfaction score, the AI should ask:

1. Did today's plan feel realistic?

2. Are you satisfied with today's progress?

3. If you could change one thing about today, what would it be?

The AI should ask additional follow-up questions only when necessary.

Example:

If today's plan was unrealistic:

"What made it unrealistic?"

* Too many tasks
* Unexpected events
* Low energy
* Underestimated work
* Other

The AI should adapt tomorrow's plan based on these responses.

---

## 3. Dynamic Reflection Length

Reflection should adapt to the day.

Normal day:

* 2–3 questions

Difficult day:

* Additional follow-up questions

The AI should avoid asking unnecessary questions to reduce reflection fatigue.

---

## 4. Design Principles

The Long Run should always follow these principles:

* Never manipulate users.
* Never use fake motivation.
* Praise only when supported by evidence.
* Every question should improve tomorrow's planning.
* Build trust through accurate understanding.
* Reflection should feel like a conversation, not a form.
* Help users create sustainable habits rather than maximize productivity.
